mato74 (Marten Schoonman) November 22, 2016, 4:57pm #2

Hi Sarah, great diagram! Very helpful I think to see both the possibilities and the relationships.

I suggest to add Akvo RSR (really simple reporting) to the scheme. Using a generic link to an xml file, one can add or update project data in the Akvo RSR project editor and create a new IATI file which is hosted on the RSR server and immediately availed through the generic link.

More information on how this works is captured here: http://rsrsupport.akvo.org/article/show/how-can-i-publish-to-iati.

SJohns (SJohns) November 22, 2016, 5:18pm #3

Hi Marten, thank you for this information. So as I understand it, organisations use Akvo RSR to create the XML file and an URL (ie http://rsr.akvo.org/organisation/42/iati/), and then they manually copy and paste the link to the file into their Registry account using the Add Datafile option. Is that right?

mato74 (Marten Schoonman) November 24, 2016, 10:28am #4

That’s correct. And when an updated IATI file is created (and hosted in RSR), it is immediately available through the general link.
